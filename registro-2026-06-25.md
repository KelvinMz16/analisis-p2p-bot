### 5. Solución final: `run.py` wrapper
- **Archivo nuevo**: `run.py` — wrapper que:
  - Inicia health servers multi-thread en puertos 8080 y 7860
  - Ejecuta `bot_p2p.py` como subprocess
  - Si el bot crashea, lo reinicia en 5s
  - El proceso padre (`run.py`) **nunca muere** (while True infinito)
- **`bot_p2p.py`**: Se removió health server interno (evita conflicto de puertos)

## Causa raíz identificada
El proceso `bot_p2p.py` muere después de 1-2 ciclos de monitoreo (causa exacta no determinada — posible OOM, excepción no capturada en `time.sleep(60)` fuera del try/except, o señal externa). Al morir, el contenedor se cae y HF Spaces lo reinicia en bucle.

La solución fue desacoplar el health server del bot usando `run.py` como proceso padre que:
1. Mantiene el health server vivo independientemente del estado del bot
2. Mantiene el contenedor vivo (while True infinito)
3. Reinicia el bot automáticamente si crashea
